재미없는 프로젝트의 문제점


디자인 철학이 없음 : 누구를 대상으로 만든건지, 어떤  시나리오를 예상하는지?

평가가 없음 -> 사람들이 진짜 좋아하는지

평범한거 하지 마라!
AI가 다 해주는 세상에서 기능 막 넣고, 인용 넣고 그런거 진짜로 의미가 하나도 없다.
퀄리티가 중요하다.


## 재미있는 프로젝트

뭔가를 빼는데 노력한 프로젝트 -> 더하는 것보다 빼는게 어렵다

뭔가를 곱하는데 노력해야한다 -> A분야 x B분야 -> 잡식성으로 먹어라

What이 아니라 Why에 집중해라
내가 왜 이 기업에 지원했고 왜 이걸 했는지 등등


객관적인 것보다 주관적인 것을 담아라 도발적인 아이디어!

내가 좋다고 생각하는 것을 남에게 보여줘야 한다!

객관적인거는 국어사전이다 -> 국어사전 재밌냐?
너 글도 재밌냐? 너만의 철학이 있냐?

사람들이 안쓰는데는 이유가 있음 -> 이유를 찾아봐라(이런 방식으로 가지 않으려고 노력해야함)


개선은 문제의 원인을 찾는 것부터 시작한다
진짜 문제는 표면적인 기능보다는 인간에게 있을 때가 많다. -> Human Centered로 접근하는 방식!!!

기말고사 후반부 내용에 집중해라
